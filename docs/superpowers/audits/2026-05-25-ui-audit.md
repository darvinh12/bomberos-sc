**Archivos / líneas:**

**Descripción.** Combinaciones `bg-green-100 text-green-800`, `bg-yellow-100 text-yellow-800`, `bg-red-100 text-red-800`, `bg-blue-100 text-blue-800`, `bg-gray-200 text-gray-700` son pensadas para fondos blancos. ### H02 — Hub `/admin` con emojis grandes y sin agrupación (P0)

**Archivo:** `apps/web/src/app/(app)/admin/page.tsx`

**Descripción.** 9 cards con emoji 3xl + título + descripción en un grid plano 3×3. Choca explícitamente con la regla "no fitness-app" del `PRODUCT.md` ("nada de iconos circulares de colores brillantes"). Además no comunica jerarquía cuando hay 9 ítems.

**Fix.**
- Agrupar en 3 secciones explícitas: **Seguridad y accesos** (Usuarios, Roles, Matriz de permisos), **Estructura organizacional** (Departamentos, Catálogos, Campos personalizados), **Configuración del sistema** (Parámetros, Módulos, Auditoría).
- Reemplazar emojis por iconos Lucide (`UserCog`, `Lock`, `LayoutGrid`, `Building2`, `BookMarked`, `ClipboardList`, `SlidersHorizontal`, `Boxes`, `BookOpen`).
- Icono pequeño (`size-5`) en una esquina del card, no protagonista. Sin colores brillantes; solo `text-primary` sutil.

**Esfuerzo.** M.

- `text-slate-500` → `text-muted-foreground`
- `text-slate-600` → `text-muted-foreground/70`
- `bg-slate-700` (foto placeholder) → `bg-muted`

**Esfuerzo.** M — bastantes reemplazos pero mecánicos.

### H08 — `next/font` no declarado (P1)

**Archivo:** `apps/web/src/app/layout.tsx`

**Descripción.** No hay declaración explícita de familia tipográfica con `next/font`. El resultado:
- macOS Safari muestra San Francisco.
- Windows Chrome muestra Segoe UI.
- Linux Firefox cae a DejaVu.

Los anchos varían y los `tabular-nums` quedan mejor con Inter. El sistema institucional debe tener tipografía consistente.

**Fix.** Importar `Inter` de `next/font/google` con `subsets: ['latin']`, `display: 'swap'`, `variable: '--font-inter'` y aplicar `className={inter.variable}` en `<html>`. Actualizar `globals.css` para usar `var(--font-inter)` como `font-family` del body.

**Esfuerzo.** S.

### H09 — Falta skip link (P1)

**Archivo:** `apps/web/src/app/(app)/layout.tsx`

**Descripción.** Un usuario con teclado debe tabular toda la sidebar (12+ links) antes de llegar al contenido. Skip link estándar de accesibilidad falta.

**Fix.** Añadir al inicio del shell:
```jsx
<a href="#main-content"
   className="sr-only focus:not-sr-only focus:fixed focus:top-2 focus:left-2 focus:z-50 focus:px-4 focus:py-2 focus:bg-primary focus:text-primary-foreground focus:rounded">
  Saltar al contenido
</a>
```
Y `<main id="main-content" tabIndex={-1}>`.

**Esfuerzo.** S.

### H10 — Focus visible inconsistente (P1)

**Archivo:** múltiples — `apps/web/src/app/globals.css` (solo `.input`), todos los `<button>` y `<Link>` confiando en el default del navegador.

**Descripción.** En navy oscuro, el outline default del navegador (azul brillante) se ve mal y poco accesible. Necesitamos focus-visible deliberado.

**Fix.** Añadir reglas `@layer base` en `globals.css`:
```css
@layer base {
  *:focus-visible {
    outline: 2px solid hsl(var(--ring));
    outline-offset: 2px;
    border-radius: 2px;
  }
}
```
Y opcionalmente, una utility `.focus-ring` que componentes interactivos pueden aplicar para refinar.

**Esfuerzo.** M — el cambio global es 4 líneas, pero hay que verificar que ningún focus se ve mal.

### H17 — Falta sistema de clases utility (P1)

**Archivo:** `apps/web/src/app/globals.css`

**Descripción.** El patrón `rounded-md bg-primary text-primary-foreground px-4 py-2 text-sm font-medium hover:opacity-90` se repite en 40+ puntos. Igual para el botón secundario y para badges. Esto introduce drift visual (alguien copia mal en una página y queda 4px distinto).

**Fix.** En `@layer components`:
```css
.btn-primary { @apply inline-flex items-center justify-center rounded border border-transparent bg-primary text-primary-foreground px-4 py-2 text-sm font-medium hover:opacity-90 disabled:opacity-60 transition-opacity focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 focus-visible:ring-offset-background; }
.btn-secondary { @apply inline-flex items-center justify-center rounded border border-border bg-transparent text-foreground px-4 py-2 text-sm font-medium hover:bg-accent transition-colors focus-visible:ring-2 focus-visible:ring-ring; }
.btn-ghost { @apply inline-flex items-center justify-center rounded text-muted-foreground hover:text-foreground hover:bg-accent px-3 py-1.5 text-sm transition-colors; }
.badge { @apply inline-flex items-center px-2 py-0.5 rounded text-xs font-medium border; }
.badge-success { @apply border-emerald-700/50 bg-emerald-900/40 text-emerald-400; }
.badge-warning { @apply border-amber-700/50 bg-amber-900/40 text-amber-400; }
.badge-info { @apply border-blue-700/50 bg-blue-900/40 text-blue-400; }
.badge-danger { @apply border-red-700/50 bg-red-900/40 text-red-400; }
.badge-neutral { @apply border-border bg-muted text-muted-foreground; }
.badge-primary { @apply border-primary/30 bg-primary/15 text-primary-foreground/80; }
```

Y luego migrar las pantallas críticas (login, layout, dashboard, funcionarios, admin hub) en esta tanda. El resto migra en una pasada incremental.

**Esfuerzo.** M.
